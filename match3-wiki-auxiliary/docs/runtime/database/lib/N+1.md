# N+1 查询问题

**N+1 查询问题（N+1 Query Problem）** 是 ORM 使用中最常见的性能陷阱：查询 1 次获取 N 条父记录后，对每条父记录各自发出 1 次子查询，总计产生 N+1 次数据库请求。

## 典型场景

加载一批文件及其对应的 TextChunk 数量：

```python
# BAD: N+1 queries
files = await session.execute(select(RawFile).limit(20))
for file in files.scalars():
    # Each iteration triggers a separate SELECT to load chunks — 20 queries!
    chunks = await session.execute(
        select(TextChunk).where(TextChunk.raw_file_id == file.id)
    )
    print(len(chunks.scalars().all()))
```

20 条文件记录 → 1（获取文件）+ 20（每条各查 chunks）= **21 次查询**。

## 解决方案

使用预加载（Eager Loading）一次性取回关联数据：

```python
# GOOD: 2 queries total (selectinload)
from sqlalchemy.orm import selectinload

result = await session.execute(
    select(RawFile)
    .options(selectinload(RawFile.chunks))
    .limit(20)
)
files = result.scalars().all()
# All chunks are already loaded, no extra queries
for file in files:
    print(len(file.chunks))
```

两种预加载策略的详细对比见 [loading/selectinload.md](./loading/selectinload.md) 和 [loading/joinedload.md](./loading/joinedload.md)。

## 识别 N+1

在开发环境中可以通过 SQLAlchemy 的 SQL 日志发现：

```python
import logging
logging.getLogger("sqlalchemy.engine").setLevel(logging.INFO)
# Will print every SQL statement executed — spot repeated queries
```

若看到循环体内反复出现相似的 `SELECT` 语句，通常就是 N+1 问题。
