# BM25

**BM25（Best Match 25，最佳匹配 25）** 是信息检索领域最经典的关键词相关性打分算法，由 Robertson 等人在 1994 年提出。它基于概率检索模型，综合考虑词频、逆文档频率和文档长度，计算查询词与文档的相关性分数。Elasticsearch 默认使用 BM25 作为全文检索的排序算法。

## 公式

$$\text{BM25}(q, d) = \sum_{t \in q} \text{IDF}(t) \cdot \frac{f(t,d) \cdot (k_1 + 1)}{f(t,d) + k_1 \cdot (1 - b + b \cdot \frac{|d|}{\text{avgdl}})}$$

各符号含义：

| 符号 | 含义 |
|------|------|
| $t$ | 查询中的词项（Token） |
| $f(t,d)$ | 词项 $t$ 在文档 $d$ 中的词频（TF） |
| $\text{IDF}(t)$ | 词项 $t$ 的逆文档频率，越稀有分越高 |
| $\|d\|$ | 文档长度（词数） |
| $\text{avgdl}$ | 所有文档的平均长度 |
| $k_1$ | 词频饱和系数，通常取 1.2–2.0 |
| $b$ | 长度惩罚系数，通常取 0.75 |

直觉上：词频越高分越高，但有上限（饱和）；文档越长会被惩罚（防止靠堆砌关键词得高分）；词越稀有贡献越大。

## 在 Elasticsearch 中的使用

Elasticsearch 的 `match` 查询默认使用 BM25：

```json
{
  "query": {
    "match": {
      "content": "match3 retention strategy"
    }
  }
}
```

返回结果的 `_score` 字段即为该文档的 BM25 分数。

## BM25 vs 向量检索

| 能力 | BM25 | 稠密向量检索 |
|------|------|-------------|
| 精确关键词匹配 | ✓ 强 | ✗ 弱 |
| 语义相似度（同义词、改写） | ✗ 弱 | ✓ 强 |
| 跨语言检索 | ✗ | ✓（多语言模型） |
| 无需训练 | ✓ | ✗ 需要嵌入模型 |

本项目混合检索中，BM25（通过 Elasticsearch）与稠密 + 稀疏向量（通过 Milvus）三路并行，再用 RRF 融合，两者优势互补。
