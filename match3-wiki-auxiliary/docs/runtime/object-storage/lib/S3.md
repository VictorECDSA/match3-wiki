# S3 API 兼容性

**S3 API** 是 Amazon Simple Storage Service（简单存储服务）定义的 HTTP 对象存储接口规范，已成为对象存储领域的事实标准。MinIO 完整兼容 S3 API，这意味着所有为 AWS S3 编写的客户端代码都可以直接对接 MinIO，只需替换端点地址（Endpoint）。

## 为什么选择 S3 兼容而非自有 API

- **可移植性**：代码写一次，可在 MinIO（本地/私有云）和 AWS S3（公有云）之间无缝切换
- **生态成熟**：大量开源工具（boto3、MinIO Python SDK、s3fs 等）直接可用
- **无供应商锁定**：底层存储可以替换，业务代码不感知

## 核心操作映射

| S3 API 操作 | MinIO Python SDK | 说明 |
|-------------|-----------------|------|
| `PutObject` | `put_object` / `fput_object` | 上传对象 |
| `GetObject` | `get_object` / `fget_object` | 下载对象 |
| `DeleteObject` | `remove_object` | 删除对象 |
| `ListObjectsV2` | `list_objects` | 列举对象 |
| `CreateBucket` | `make_bucket` | 创建存储桶 |
| `CreateMultipartUpload` | 自动（`fput_object` 超阈值触发） | 分片上传 |

## 本项目的端点配置

```python
from minio import Minio

client = Minio(
    endpoint="localhost:9000",           # MinIO server (not AWS)
    access_key=env.MINIO_ACCESS_KEY,
    secret_key=env.MINIO_SECRET_KEY,
    secure=False,                         # TLS off for local dev, on for production
)
```

切换到 AWS S3 时：

```python
client = Minio(
    endpoint="s3.amazonaws.com",
    access_key=env.AWS_ACCESS_KEY_ID,
    secret_key=env.AWS_SECRET_ACCESS_KEY,
    secure=True,
    region="ap-east-1",
)
```

业务代码（`ObjectStorage` Protocol 实现层）完全相同，只需在 `build_runtime()` 中修改连接参数。
