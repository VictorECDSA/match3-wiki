# 近似最近邻（ANN）

**近似最近邻（ANN，Approximate Nearest Neighbor）** 是向量检索的核心算法类别。与精确最近邻（Exact Nearest Neighbor）搜索相比，ANN 以极小的精度损失换取数量级级别的速度提升。

## 为什么需要"近似"

精确最近邻需要将查询向量与数据库中所有向量逐一计算距离，时间复杂度为 O(N×d)（N 为向量数，d 为维度）。当 N 达到百万级、d 达到千级时，每次检索耗时数秒，不可接受。

ANN 通过预建索引结构（图、树、聚类等）将搜索范围缩小到候选子集，时间复杂度降至近似 O(log N)，但召回率（Recall）从 100% 降至 95–99%——对大多数检索场景已足够。

## 召回率 vs 速度的权衡

ANN 索引都提供一个"努力程度"参数来调节这一权衡：

| 索引 | 参数 | 更高值的效果 |
|------|------|--------------|
| HNSW | `ef` | 召回率更高，速度更慢 |
| IVF_FLAT | `nprobe` | 召回率更高，速度更慢 |
| DiskANN | `search_list` | 召回率更高，磁盘读取更多 |

## 评估指标

**召回率@K（Recall@K）**：在真实 top-K 结果中，ANN 检索到的比例。本项目通过混合检索（Hybrid Search）+ 重排序（Rerank）两道过滤进一步提升最终质量，对 ANN 本身召回率要求适当放宽。

## 本项目的索引选择

默认使用 [HNSW](./indexes/HNSW.md)（内存索引，高召回率，低延迟）。超出内存时切换至 [DiskANN](./indexes/DiskANN.md)。
