# BM42

**BM42** 是由 Qdrant 提出的稀疏向量生成算法，将经典的 BM25（Best Match 25）统计方法与 Transformer 注意力机制结合，生成兼具关键词精确性和语义感知能力的稀疏向量。

## 与 BM25 的关系

传统 BM25 使用词频（TF）和逆文档频率（IDF）计算词项权重，权重由统计规律驱动，不理解语境。BM42 将 BM25 的 TF 部分替换为 Transformer 对该词的注意力分数：注意力高的词（语境中更重要的词）获得更高权重，而非仅凭出现频率。

```
BM25 weight = IDF(t) × TF_normalized(t, d)
BM42 weight = IDF(t) × Attention_score(t, d)  # attention from a lightweight Transformer
```

## 输出格式

BM42 输出一个稀疏向量，维度等于词表大小（数万维），但绝大多数维度为零：

```python
# BM42 sparse vector example
{
    15234: 0.81,   # high-attention token in this context
    42891: 0.63,
    7102:  0.35,
}
```

在 Milvus 中存储为 `SPARSE_FLOAT_VECTOR` 类型。

## 使用场景

BM42 稀疏向量配合稠密向量（Dense Vector）在同一 Collection 中双路检索，再通过 RRF 融合——这是本项目混合检索（Hybrid Search）的核心构成。详见 [sparse.md](./vectors/sparse.md) 和 [RRF.md](./RRF.md)。
