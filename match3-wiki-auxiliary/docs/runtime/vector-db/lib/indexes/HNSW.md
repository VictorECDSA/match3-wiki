# HNSW 索引

**HNSW（Hierarchical Navigable Small World，分层可导航小世界图）** 是当前最主流的近似最近邻（ANN）向量索引算法，由 Malkov 和 Yashunin 于 2016 年提出。

## 工作原理

HNSW 将向量组织成多层图结构：

- **底层（Layer 0）**：包含所有向量节点，每个节点与最近邻形成边
- **高层（Layer 1、2…）**：向上逐层稀疏采样，节点数呈指数减少
- **搜索路径**：从最高层的入口节点出发，贪心地向目标向量靠近，到达底层后精排

这与"六度分隔"理论中的小世界网络（Small World Network）结构类似，高层负责远距离跳跃定位，底层负责精细检索。

## 关键参数

| 参数 | 含义 | 典型值 |
|------|------|--------|
| `M` | 每个节点在底层保留的最大邻居数 | 16 |
| `efConstruction` | 构建索引时的搜索候选集大小，越大索引质量越高但构建越慢 | 200 |
| `ef` | 查询时的候选集大小，越大召回率越高但速度越慢 | 100 |

```python
index_params = {
    "metric_type": "COSINE",
    "index_type": "HNSW",
    "params": {
        "M": 16,
        "efConstruction": 200,
    },
}
search_params = {"ef": 100}
```

## 优缺点

| 优点 | 缺点 |
|------|------|
| 查询速度快（毫秒级） | 全量数据必须驻留内存 |
| 召回率高（通常 >95%） | 索引构建内存开销大 |
| 增量插入无需重建 | — |

## 与其他索引对比

HNSW 适合数据量可完全放入内存的场景（本项目默认选择）。数据量超出内存时，改用 [DiskANN](./DiskANN.md)；追求极低内存占用时，可选 [IVF_FLAT](./IVF_FLAT.md)。
